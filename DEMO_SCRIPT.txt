[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how data flows. 
First, the raw invoice text enters the "security_checkpoint" node, which acts as our security gatekeeper.
If any prompt injection keyword is found, the workflow routes to the "security_block_node" and stops immediately.
Otherwise, the raw text proceeds to the "extractor_node", where our ExtractorAgent uses the "parse_invoice" MCP tool to extract structured data.
Next, the "validator_node" runs our ValidatorAgent, which pulls internal records using the "lookup_purchase_order" tool and runs "validate_invoice_against_po" to detect mismatches.
From there, the "approval_node" runs our ApprovalAgent to decide if the invoice can be auto-approved or if it needs human eyes.
If review is needed, the workflow routes to the "human_review_node" which pauses the execution for a manual check.
Finally, the "final_output_node" compiles the decision and prints the audit logs.
